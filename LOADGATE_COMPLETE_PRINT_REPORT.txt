WHAT IS LOADGATE?

LoadGate is a comprehensive IoT-enabled vehicle management solution that 
combines:

→ Hardware Layer (Arduino + Sensors)
   • Real-time vehicle detection using sonar
   • Multi-cell weight measurement for accuracy
   • Automated barrier control via servo motor
   • Serial communication with backend system

→ Backend Layer (Node.js/Express API)
   • RESTful API for system communication
   • Business logic and decision processing
   • External AI integration (Google Gemini)
   • Real-time event streaming (SSE)
   • MongoDB data persistence

→ Frontend Layer (Next.js/React Web App)
   • Responsive web interface
   • Real-time dashboards
   • Analytics and reporting
   • User authentication and authorization
   • Multi-format exports (PDF, Excel, Word)

THREE-TIER ARCHITECTURE MODEL:

                        ┌─────────────────┐
                        │   Frontend      │
                        │   (Next.js)     │
                        │   Web Browser   │
                        └────────┬────────┘
                                 │
                      (HTTP REST + SSE)
                                 │
                        ┌────────▼────────┐
                        │    Backend      │
                        │  (Express.js)   │
                        │  Business Logic │
                        │   Data Process  │
                        └────────┬────────┘
                                 │
                   (Serial COM + EventEmitter)
                                 │
                        ┌────────▼────────┐
                        │   Hardware      │
                        │    (Arduino)    │
                        │   Sensors &     │
                        │   Actuators     │
                        └─────────────────┘


ARCHITECTURE PRINCIPLES:

1. Separation of Concerns
   ✓ Each layer has distinct responsibility
   ✓ Clear interfaces between layers
   ✓ Independent scaling possibilities
   ✓ Easier maintenance and testing

2. Event-Driven Design
   ✓ Arduino emits sensor events
   ✓ Backend listens and processes
   ✓ Frontend subscribes to updates
   ✓ Minimal coupling between components

3. Stateless Backend
   ✓ No session storage needed
   ✓ Scales horizontally
   ✓ Fault tolerant design
   ✓ Load balancer friendly

4. Real-Time Capabilities
   ✓ SSE (Server-Sent Events) for live updates
   ✓ WebSocket option for future
   ✓ Event streaming to multiple clients
   ✓ Dashboard updates without polling

5. Security-First Approach
   ✓ Authentication at every layer
   ✓ Authorization checks on operations
   ✓ HTTPS encryption in transit
   ✓ Database encryption at rest

│ Clock Speed: 16 MHz                                 │
│ Flash Memory: 32KB                                  │
│ RAM: 2KB                                            │
│ EEPROM: 1KB                                         │
│ Digital I/O: 14 pins (6 PWM)                        │
│ Analog Input: 6 pins                                │
│ Operating Voltage: 5V                               │
│ Recommended: 7-12V input                            │
│ Current Draw: 50-100mA                              │
│ USB: Type B connector                               │
│ Development: Arduino IDE 2.0+                       │
│ Cost: $20-30                                        │
│ Purpose: Central controller for all sensors         │
└─────────────────────────────────────────────────────┘

Pin Usage:
├─ Pin 9:  Sonar TRIG (Digital Output, PWM)
├─ Pin 10: Sonar ECHO (Digital Input)
├─ Pin 11: Servo Motor (Digital Output, PWM)
├─ Pin 12: HX711 SCK/Clock (SPI Clock)
├─ Pin 13: HX711 DT/Data (SPI Data)
└─ Pins GND, 5V: Power distribution


COMPONENT 2: Load Cells (4 × 100kg)
┌─────────────────────────────────────────────────────┐
│ Quantity: 4 cells (one per vehicle wheel/axle)      │
├─────────────────────────────────────────────────────┤
│ Type: Compression/Tension Strain Gauge               │
│ Capacity (each): 100kg (220 lbs)                    │
│ Total Capacity: 400kg                               │
│ Output Type: Analog (4-wire bridge)                 │
│ Excitation Voltage: 5V DC                           │
│ Output Sensitivity: ~2mV/V                          │
│ Non-linearity: ±0.05% FS                            │
│ Hysteresis: ±0.05% FS                               │
│ Temperature Range: -10°C to +60°C                   │
│ Connection: Wired in parallel pairs:                │
│   • L1 + L3 (Left/Right): One HX711 input           │
│   • L2 + L4 (Front/Rear): Same HX711 input          │
│ Wiring: Red=+Exc, Black=-Exc, Green=+Out, White=-Out
│ Cost: $30-50 per cell ($120-200 total)              │
│ Purpose: Measure vehicle weight                     │
└─────────────────────────────────────────────────────┘

Load Cell Placement:
    Front Left (L1) ─────────┬──────── Front Right (L2)
                             │
                         Vehicle
                             │
     Rear Left (L3) ──────────┴──────── Rear Right (L4)

Parallel Configuration:
├─ L1 & L3 connected in parallel (left side)
├─ L2 & L4 connected in parallel (right side)
├─ Both pairs to single HX711 for total weight
└─ Result: 400kg total capacity measurement


COMPONENT 3: HX711 Load Cell Amplifier
┌─────────────────────────────────────────────────────┐
